Variables and Assignment

Key Rule: = means "ASSIGN" not "equals"

Assignment flows right to left

Code What Happens Result
age = 14 Store the value 14 in variable called age age contains 14
name = "Sam" Store the text "Sam" in variable called name name contains "Sam"
result = 5 + 3 Calculate 5+3, then store 8 in result result contains 8
x = x + 1 Get current value of x, add 1, store back in x x increases by 1

Variables vs Constants

Variables

  • Can be changed
  • Values can be updated
  • Example: score = 10
    Later: score = 15

Constants

  • Never change
  • Fixed values
  • Example: PI = 3.14159
    Always stays the same

Order of Operations (BODMAS)

2 + 3 * 4 = 14 (multiplication first)
(2 + 3) * 4 = 20 (brackets first)
10 / 2 + 3 = 8.0 (division first)
15 % 4 * 2 = 6 (left to right for same precedence)

Functions and Return Values

Some functions return values you can store, others don't!

Function What it Does Returns Example
input() Gets text from user String name = input("Enter name: ")
int() Converts to integer Integer age = int("14")
float() Converts to real number Real (float) height = float("5.8")
str() Converts to string String text = str(100)
print() Displays text None (nothing useful) print("Hello")

Important Distinction:

  • Functions that return useful values: input(), int(), float(), str() - you can assign these to variables
  • Functions that don't return useful values: print() - these do something but don't give you data back
  • Example: x = print("Hello") would store None in x (not useful!)

Combining Functions

Step by Step:

user_text = input("Enter a number: ")    # Returns string
number = int(user_text)                   # Returns integer
result = number * 2                       # Calculate
print(f"Double is {result}")              # Display result

Combined (Advanced):

number = int(input("Enter a number: "))  # Function inside function!
print(f"Double is {number * 2}")          # Calculate and display

Input and Output

Getting Input

  • input("prompt") - Always returns string
  • int(input("prompt")) - Convert to integer
  • float(input("prompt")) - Convert to real

Examples:

name = input("Name? ")
age = int(input("Age? "))
height = float(input("Height? "))

Displaying Output

  • print("text") - Print text
  • print(variable) - Print variable
  • print("text", variable) - Print both
  • print(f"text {variable}") - F-strings

Examples:

print("Hello!")
print(name)
print("Hi", name)
print(f"Hi {name}")

Type Casting

Why? Because input() always returns strings!

From To Function Example
String Integer int() int("42") → 42
String Real float() float("3.14") → 3.14
Integer String str() str(42) → "42"
Real String str() str(3.14) → "3.14"
Real Integer int() int(3.8) → 3 (truncated!)

Common Pattern:

# User types "25"
age_text = input("Enter age: ")     # age_text = "25" (string)
age_number = int(age_text)          # age_number = 25 (integer)
next_year = age_number + 1          # next_year = 26 (math works!)
print(f"Next year: {next_year}")    # Output: Next year: 26

Complete Program Example

Simple Calculator

# Get user information
name = input("What's your name? ")
print(f"Hello {name}! Let's do some math.")

# Get numbers from user (with type casting)
num1 = float(input("Enter first number: "))
num2 = float(input("Enter second number: "))

# Calculate results (arithmetic operators)
addition = num1 + num2
subtraction = num1 - num2
multiplication = num1 * num2

# Display results (formatted output)
print(f"{num1} + {num2} = {addition}")
print(f"{num1} - {num2} = {subtraction}")
print(f"{num1} * {num2} = {multiplication}")
print(f"Thanks for using the calculator, {name}!")

This Program Shows:

  • String and Real data types
  • Variable assignment
  • Functions with return values
  • Type casting with float()
  • Arithmetic operators
  • Input → Process → Output flow

Memory Connection

From Your Code to Computer Memory:

1. You write: age = 14
2. Python evaluates: 14
3. Python converts: 14 → binary 00001110
4. Python stores: Binary in 4 bytes of RAM
5. Python creates: Label "age" pointing to that memory

Remember: All your data (numbers, text, booleans) eventually becomes binary in computer memory - connecting back to your Boolean logic and binary knowledge!

  • Casting = Conversion: Changing one type of container to another
